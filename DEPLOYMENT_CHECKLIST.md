## Render.com Deployment (Recommended)

### Step 1: Prepare GitHub Repository

```bash
# Ensure you're on main branch
git branch
# Should see: * main

# Review recent changes
git log --oneline -5

# Check file status
git status
# Should show: working tree clean

# If not, commit changes
git add -A
git commit -m "Audit: Fix duplicates and clean configuration"
```

### Step 2: Push to GitHub

```bash
git push origin main
```

### Step 3: Connect to Render

1. **Log in to Render Dashboard**
   - URL: https://dashboard.render.com
   - Sign in with GitHub account

2. **Create New Service**
   - Click "New +" button (top right)
   - Select "Blueprint"
   - Click "Connect GitHub"
   - Authorize if prompted

3. **Select Repository**
   - Find: `jshkbt-create/Maxus`
   - Click "Connect"
   - Render will auto-detect `render.yaml`

4. **Review Services**
   - You'll see:
     - `maxus-api` (Web Service)
     - `maxus-db` (PostgreSQL Database)
   - Verify settings match below

### Step 4: Configure Environment Variables

Click on **`maxus-api` service** and set:

#### Required (Choose One)

**Option A: OpenAI**
```
OPENAI_API_KEY=sk-your-key-here
LLM_PROVIDER=openai
LLM_MODEL=gpt-4-mini
```

**Option B: Anthropic**
```
ANTHROPIC_API_KEY=sk-ant-your-key-here
LLM_PROVIDER=anthropic
LLM_MODEL=claude-3-haiku-20240307
```

#### Optional

```
NEWS_API_KEY=your-newsapi-key
ALLOWED_ORIGINS=["https://yourdomain.com"]
```

> **Note**: `SECRET_KEY`, `DATABASE_URL`, and `REDIS_URL` are auto-generated by Render

### Step 5: Deploy

1. Review all settings
2. Click **"Deploy"** button
3. Monitor deployment logs (should take 5-10 minutes):
   - ✅ Build starting
   - ✅ Dependencies installing
   - ✅ Application building
   - ✅ Database initializing
   - ✅ Service starting

### Step 6: Verify Deployment

Once deployment completes:

```bash
# Replace YOUR_APP with actual Render URL
YOUR_APP="https://maxus-api.onrender.com"

# Test health endpoint
curl ${YOUR_APP}/health
# Expected: {"status":"healthy","service":"manus-agent"}

# Test docs endpoint
curl ${YOUR_APP}/docs
# Should return HTML (Swagger UI)

# Test root endpoint
curl ${YOUR_APP}/
# Should show service info
```

# Replace with your actual URL

# 1. Health endpoint
echo "Testing health endpoint..."
curl -X GET ${BASE_URL}/health

echo "Testing Swagger docs..."
curl -s ${BASE_URL}/docs | grep -q "swagger" && echo "✅ Docs available"

# 3. Simple agent task

curl -X POST ${BASE_URL}/api/v1/agent/execute \
  -H "Content-Type: application/json" \

## Troubleshooting

### Problem: Build Fails

**Check:**
1. View logs in Render dashboard
2. Verify all files are committed (`git status`)
3. Check Python version matches (3.11)
4. Verify requirements.txt has no syntax errors

**Solution:**
```bash
# Test locally
python3 -m venv test_env
source test_env/bin/activate
pip install -r requirements.txt
# If this fails, requirements.txt has issues
```

### Problem: API Returns 500 Error

**Check:**
1. View logs in Render dashboard
2. Verify environment variables are set
3. Check DATABASE_URL and REDIS_URL exist
4. Verify API keys are valid

**Solution:**
```bash
# Check env vars are set
curl https://your-app.onrender.com/health

# If fails, check Render dashboard → Service → Environment
# Verify all required vars have values (not empty)
```

### Problem: Database Connection Error

**Check:**
1. PostgreSQL service exists in Render
2. DATABASE_URL environment variable is set
3. Database service is "Available" (not suspended)

**Solution:**
```bash
# In Render dashboard:
# 1. Go to maxus-db service
# 2. Check status (should be "Available")
# 3. Go to maxus-api service
# 4. Click "Environment"
# 5. Verify DATABASE_URL is set and not empty
```

### Problem: LLM Service Not Working

**Check:**
1. API key is set in environment
2. API key is valid (test in API provider console)
3. Account has sufficient credits
4. API key has correct permissions

**Solution:**
```bash
# Test API key validity
# OpenAI
curl https://api.openai.com/v1/models \
  -H "Authorization: Bearer sk-your-key"

# Anthropic
curl https://api.anthropic.com/v1/models \
  -H "x-api-key: sk-ant-your-key"
```

### Problem: Timeout Errors

**Check:**
1. Agent tasks are taking too long
2. LLM API is slow
3. Database queries are slow

**Solution:**
```bash
# Check logs for slow operations
# Render dashboard → maxus-api → Logs → Search for "timeout"

# Increase timeout (if using Render):
# Edit render.yaml → add: timeoutSec: 300
```

## Rollback Procedure

If issues occur after deployment:

### Quick Rollback

```bash
# Disable service without deleting
# Render Dashboard → Service → Settings → Disable Service

# Check previous deployment
# Render Dashboard → Service → Deployments → Select previous version
# Click "Redeploy"
```

### Full Rollback

```bash
# Revert last commit
git revert HEAD
git push origin main

# Render will auto-deploy previous version
```
